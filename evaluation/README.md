# Evaluation

This directory contains KPI evaluation scripts and results for all project phases.

## Directory Structure

```
evaluation/
├── phase3_comparison.py              # Phase 3 routing comparison
├── phase3_kpi_regression_gate.py     # Regression testing for routing
├── hackathon_kpi/                    # Raw SUMO output files
│   ├── *.tripinfo.xml                # Per-vehicle trip information
│   ├── *.statistics.xml              # Aggregate statistics
│   └── *.summary.xml                 # Simulation summaries
└── *.json                            # KPI result files
```

## Evaluation Scripts

### Phase 3: Routing Comparison

Compare risk-aware routing vs baseline:

```bash
python3 evaluation/phase3_comparison.py
```

**Output**: `phase3_comparison_results.json`

### Phase 3: KPI Regression Gate

Ensure routing changes don't degrade performance:

```bash
python3 evaluation/phase3_kpi_regression_gate.py
```

**Output**: `phase3_kpi_regression_results_final.json`

### Phase 4 & 5: Results Only

Phase 4 (RL) and Phase 5 (Fusion) results are generated by their respective training/ablation scripts and stored here.

## Key Performance Indicators

### Phase 3: Routing KPIs

| Metric | Description | Target |
|--------|-------------|--------|
| Reroute Count | Number of vehicles rerouted | Informational |
| Congestion Avoidance Rate | % of predicted congestion avoided | ≥ 70% |
| Travel Time Delta | Change vs baseline | ≤ +5% (no worse) |
| Risk Score Accuracy | Predicted vs actual congestion | ≥ 80% |

### Phase 4: Signal Control KPIs

| Metric | Description | Target |
|--------|-------------|--------|
| Average Wait Time | Mean wait at signals (s) | ↓ 15% vs fixed |
| Throughput | Vehicles/hour | ↑ 10% vs fixed |
| Average Queue Length | Vehicles waiting | ↓ 20% vs fixed |
| Total Delay | Cumulative delay (s) | ↓ 15% vs fixed |

### Phase 5: Fusion KPIs

| Metric | Description | Target |
|--------|-------------|--------|
| Combined Efficiency | Routing + Signal synergy | ↑ 25% vs baseline |
| Conflict Resolution | Routing/Signal agreement rate | ≥ 90% |
| Latency | Decision time (ms) | ≤ 50ms |


## Running Evaluations

### Quick Evaluation

```bash
# Run Phase 3 comparison
python3 evaluation/phase3_comparison.py

# Check results
cat evaluation/phase3_comparison_results.json | python3 -m json.tool
```

### Full Evaluation Suite

```bash
# Phase 3
python3 evaluation/phase3_comparison.py
python3 evaluation/phase3_kpi_regression_gate.py

# Phase 4 (requires trained model)
python3 controllers/rl/train_phase4.py --profile smoke --eval-only

# Phase 5
python3 controllers/fusion/run_ablation.py --all
```

## SUMO Output Files

The `hackathon_kpi/` directory contains raw SUMO XML outputs:

- **tripinfo.xml**: Per-vehicle metrics (departure, arrival, wait times)
- **statistics.xml**: Aggregate network statistics
- **summary.xml**: Per-step simulation summaries

These are parsed by evaluation scripts to compute KPIs.
